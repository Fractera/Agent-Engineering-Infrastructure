# `_shared-v2` — мягкий дев/админ-слой автоматизаций v2

Здесь живёт **одна копия на все автоматизации** того, что нужно ТОЛЬКО во время разработки: кнопки
«Строить вместе с ИИ», админ-настройки компонентов, генератор кода, служебные функции. Это **мягкий
слой** закона устойчивости (шаг 298): «продакшн твёрдый, разработка мягкая».

## Закон устойчивости

- **Твёрдый слой (в папке автоматизации, рантайм/продакшн):** ядро (`_data/`), функции узлов
  (`_lib/nodes/`), публичные компоненты (`_components/<tab>/public`) и их рантайм-функции
  (`_lib/components/`). Зависит только от `zod` + node-builtins, запускается сам по себе, уезжает
  ZIP-архивом. **Ничего из `_shared-v2` ему не требуется.**
- **Мягкий слой (здесь, `_shared-v2`, дев-время):** дев/админ-инструменты. Папка автоматизации ссылается
  на них **за fail-silent границей**.
- **🔒 Инвариант:** продакшн НИКОГДА не зависит от жизни дев-слоя. Сломан/отсутствует `_shared-v2` — дев-
  кнопки просто не появляются, а автоматизация продолжает работать для конечного пользователя.

## Как подключаться

Единственный разрешённый внешний путь папки автоматизации — `_shared-v2`, и только из её fail-silent
дев-слота (`_components/shared/dev-slot.tsx` + `dev-slot.client.tsx`): серверный гейт по присутствию слоя
+ клиентский `next/dynamic` (`ssr:false`, `loading:()=>null`) за React error-boundary с null-фолбёком.

Публичные (рантайм) компоненты автоматизации `_shared-v2` **не импортируют вовсе** — это проверяет
`scripts/check-entity-imports.mjs` (единственный разрешённый внешний путь = `_shared-v2`, и только в
файлах дев-слота).

Импортировать из барреля `index.ts` (стабильный интерфейс), а не из внутренних файлов.
