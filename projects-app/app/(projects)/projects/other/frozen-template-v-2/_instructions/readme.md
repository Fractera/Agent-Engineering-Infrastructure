# Системные инструкции

Здесь лежат сами тексты законов — по файлу на имя. Объект в ядре несёт **имя** своей инструкции
(`systemInstructionName`), а не её текст: связь видна прямо в объекте, а платится за неё одно слово.

```
_instructions/
  passport.md              ← стартовая: как здесь вообще работать (двери, порядок итераций)
  fracteraPro.md
  graph.md
  nodes.md                 ← для ВСЕХ узлов, какого бы вида они ни были
  group.input.md · group.middle.md · group.output.md
  kind.input.md · kind.input-connector.md · kind.transform.md
  kind.condition-success.md · kind.condition-failure.md
  kind.output.md · kind.output-connector.md
  components.md
  tab.md                   ← для каждой вкладки
  useCases.md
  history.md
```

**Один источник истины.** Текст живёт только здесь. Схема знает лишь СПИСОК допустимых имён и требует от
каждого объекта своё: узел обязан нести `nodes`, вкладка — `tab`, входная группа — `group.input`. Подменить
имя нельзя — это закон, просто дешёвый.

**Как читает агент.** Либо файлом (`_instructions/<имя>.md`), либо дверью
`GET api/instruction?name=<имя>` — что дешевле в его положении. Двери, отдающие объекты (`api/work`,
`api/core`), подставляют текст сами, когда он нужен для работы.

**Как правит владелец.** Обычный markdown: открыть файл, написать, сохранить. Ни пересборки, ни правки кода.
Пустой файл — законная ситуация: инструкция ещё не написана.
