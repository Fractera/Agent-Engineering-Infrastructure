# Публичная половина Notification — намеренно пуста

У сущности «Notification» публичной половины нет: полоса-уведомление показывает состояние сборки
автоматизации (что не построено, какие предупреждения оставил агент, какие кейсы ещё новые) — это
внутренняя правда для владельца, посетителю витрины её показывать нечего.

Папка `public/` сохранена **специально пустой**, чтобы у Notification была та же единая раскладка
`public/ + admin/`, что и у всех остальных сущностей v2 (эталон — `calendar/`). Так разделение
административного и публичного слоёв остаётся ОДНОЙ системой на все сущности, а не особым случаем.

Весь код полосы живёт в `../admin/`, маршрутизатор — в `../index.tsx`.
